**SKILL.md:**
```markdown
---
name: orchestrate
description: Run multi-step tasks via claude -p. Supports inline goals, YAML job files, and status tracking.
---

# Orchestrate

Run, monitor, and manage multi-step orchestrated tasks. Each task is decomposed into sequential subtasks executed via claude -p with automatic checkpointing.

## When to Use

- Running a complex multi-step task that needs LLM reasoning at each step
- Checking status of running or completed orchestration tasks
- Canceling a stuck orchestration run

## CLI Reference

### orchestrate run <target>
Run an orchestration task. Target can be:
- A YAML file path: `orchestrate run /etc/nixos/zeroclaw/cron/jobs/daily-briefing.yaml`
- An inline goal string: `orchestrate run "Summarize today's emails and calendar"`

For inline goals, a temporary YAML is created and passed to the engine.

### orchestrate status [parent-id]
Show status of orchestration tasks. Without ID, shows recent runs. With ID, shows subtask detail.

### orchestrate list
List all orchestration runs with their status, step count, and timestamps.

### orchestrate cancel <parent-id>
Cancel a running orchestration by marking its pending steps as cancelled.
```

**cli.ts:**
Implement the 4 subcommands. The `run` subcommand calls orchestrate.ts via subprocess (NOT import). The `status`, `list`, and `cancel` subcommands query state.db directly using initStateDb().

For `run`:
- If target is a file path (contains `/` or ends with `.yaml`), pass it directly to orchestrate.ts
- If target is a string (inline goal), create a temp YAML file with `goal:` field, pass that path
- Use Bun `$` to call: `${bunPath} run /etc/nixos/zeroclaw/bin/orchestrate.ts ${targetPath}`
- Resolve bun path at runtime: `const bunPath = Bun.which("bun") ?? "/run/current-system/sw/bin/bun"` — do NOT use `${pkgs.bun}/bin/bun` (that is Nix interpolation syntax, not valid TypeScript)

For `status`:
- Query orchestration_tasks where parent_id IS NULL (parent rows) ordered by created_at DESC
- If parent-id provided, also show child rows for that parent

For `list`:
- Same as status without ID — show recent parent rows

For `cancel`:
- UPDATE orchestration_tasks SET status='cancelled', updated_at=now WHERE parent_id=? AND status IN ('pending','running')
- Also update parent row status='cancelled'

Output: JSON to stdout per skill contract. Exit 0/1.

Accept optional `--db-path` flag for test isolation (same pattern as other programs).

```markdown
## Agent Job YAML Schema

Agent jobs use `type: agent` to trigger automatic command generation. You write the goal and optional steps — `cron-sync` generates the `orchestrate.ts` command automatically.

### Schema

\```yaml
name: "Human-readable unique name"        # required
schedule: "0 7 * * *"                      # required — 5-field cron
tz: "America/Lima"                         # optional (default: UTC)
type: agent                                # required for agent jobs
goal: "Describe what this job should do"   # required — the task objective
steps:                                     # optional — explicit subtask list
  - "Step 1: do this first"
  - "Step 2: then do this"
  - "Step 3: finally do this"
notify: "+51926689401"                     # optional — WhatsApp notification on completion/failure
# DO NOT add command: — cron-sync auto-generates it
\```

### How It Works

1. `cron-sync` detects `type: agent` in the YAML file
2. It generates a shell command: `bun run /etc/nixos/zeroclaw/bin/orchestrate.ts <yaml-path>`
3. The command is registered as a normal shell job via `zeroclaw cron add`
4. When the cron fires, `orchestrate.ts` reads the YAML, decomposes the goal into steps, and executes each via `claude -p`
5. Progress is checkpointed to `state.db` — if the job fails, it resumes from the last step on the next run (within 4 hours)

### Example: Daily Email Summary

\```yaml
name: "Daily email summary"
schedule: "0 8 * * 1-5"
tz: "America/Lima"
type: agent
goal: "Summarize unread emails from the last 12 hours and send a digest"
steps:
  - "List unread emails from the last 12 hours using the email skill"
  - "Group emails by sender and priority, summarize key action items"
  - "Send the digest via notify to +51926689401"
notify: "+51926689401"
\```
```
