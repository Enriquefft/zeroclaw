## Getting Started

### Prerequisites

- NixOS with flakes enabled
- A ZeroClaw account or self-hosted instance
- Home Manager configured

### Using this as a Reference

This repo is primarily a reference — study `module.nix` to see how ZeroClaw integrates with NixOS, and `documents/` to see how identity documents shape agent behavior. You can adapt any part for your own configuration.

### Adapting for Your Own Setup

If you want to run your own ZeroClaw agent on NixOS:

1. Add ZeroClaw to your flake inputs (see `module.nix` for the input name)
2. Import the module in your home-manager config
3. Replace the identity documents in `documents/` with your own
4. Adapt `module.nix` — remove personal infrastructure (see Personalizing below)

## Key Design Decisions

Brief list (4-5 bullets) covering the non-obvious design decisions visible in this repo:
- Live documents via `mkOutOfStoreSymlink` — identity docs update without a NixOS rebuild
- Skills deploy via CLI (`zeroclaw skills install`) — not wired through Nix, because zeroclaw manages its own workspace
- `bin/` directory for scripts — zeroclaw's security policy rejects `.sh` files inside skill packages
- Multi-agent IPC via shared SQLite — agents on the same host can discover each other and exchange messages
- Config rendered at build time (`pkgs.writeText`) — config.toml is managed by Nix, not hand-edited
