**Release notes content to write:**

Title: `v1.0 — Kiro MVP`

Body:

```markdown
## What shipped

Kiro is now operational. This is the v1.0 release of the NixOS configuration that runs Kiro — Enrique's personal AI chief of staff — as a persistent agent on NixOS via ZeroClaw.

### Core capabilities

- **Persistent runtime** — ZeroClaw runs as a systemd user service, starts on boot, restarts on failure
- **WhatsApp interface** — Kiro receives and responds to messages via a Kapso WhatsApp bridge over Tailscale
- **Identity documents** — Six markdown files (IDENTITY, SOUL, AGENTS, TOOLS, USER, LORE) define who Kiro is at runtime. Live-edit — no rebuild required.
- **Skills system** — Two skills ship with v1.0:
  - `sentinel` — cron-driven health monitor that detects issues and escalates via WhatsApp
  - `repair-loop` — self-repair skill Kiro invokes when it detects a fixable problem
- **Self-modification** — Kiro can edit its own identity documents and commit the changes to git
- **Multi-agent IPC** — SQLite-backed inter-process communication so multiple ZeroClaw instances on the same host can coordinate
- **Memory** — SQLite-backed persistent memory across sessions

### What this repo contains

The complete NixOS home-manager module (`module.nix`), Kiro's actual identity documents, skill source files, cron documentation, and the CLAUDE.md operational guide that keeps coding agents and Kiro on the same page.

### Getting started

See the [README](./README.md) for prerequisites, repo structure, and how to adapt this for your own setup.

### Personal infrastructure note

This configuration includes personal infrastructure (WhatsApp bridge, Tailscale delivery, Kapso secrets) that is specific to Enrique's setup. See the **Personalizing** section of the README for what to replace when adapting this for your own use.
```
