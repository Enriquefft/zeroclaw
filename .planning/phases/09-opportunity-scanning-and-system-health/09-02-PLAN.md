**Test file (bin/self-audit.test.ts):** Write tests using bun:test with temp directories and mock SQLite databases. Each check function should be exported and independently testable. Use `mkdtempSync` for isolated filesystem state. Create mock cron DB with `bun:sqlite` for cron drift tests.

**Program (bin/self-audit.ts):** Following the sentinel-scan.ts pattern:
- Shebang: `#!/usr/bin/env bun`
- Parse `--notify <phone>` from CLI args
- Export individual check functions for testability: `checkSymlinks(map)`, `checkCronDrift(cronDbPath, yamlDir)`, `checkSkills(gitDir, deployedDir)`, `checkConfig(configPath)`
- Accept optional parameters for all paths (testability pattern from init-state-db.ts)

**Symlink check (checkSymlinks):** Takes an array of `{ source, deployed }` and returns drift items. Uses `fs.realpathSync()` to verify target. If file missing or wrong target, adds to drift list. If file is a regular file instead of symlink, flag as drift (direct edit).

**Cron drift check (checkCronDrift):** Opens cron DB read-only via `bun:sqlite`. Queries `SELECT name FROM cron_jobs WHERE name IS NOT NULL AND length(name) > 0`. Reads YAML files from the jobs directory, extracts name field via regex `^name:\s*["']?(.+?)["']?\s*$`. Compares sets. Reports extras in DB and missing from DB.

**Skills check (checkSkills):** Reads git skills directory for subdirectories containing SKILL.md. Reads deployed skills directory for subdirectories. Reports extras and missing.

**Config check (checkConfig):** Checks if `~/.zeroclaw/config.toml` exists and has size > 0. Reports drift if missing or empty.

**Main audit flow:**
1. Run all checks, collect drift items
2. Output JSON to stdout: `{ drift_count, checks: { symlinks, cron, skills, config }, items: [...] }`
3. If drift found AND --notify provided: send WhatsApp via `notify()` with urgent priority. Message format: "Self-Audit -- N drift items detected" followed by one line per item.
4. If no drift: output clean JSON, no notification
5. Always exit 0

**Default paths (overridable via function params for testing):**
- HOME: `Bun.env.HOME`
- Cron DB: `${HOME}/.zeroclaw/workspace/cron/jobs.db`
- YAML dir: `/etc/nixos/zeroclaw/cron/jobs`
- Git skills: `/etc/nixos/zeroclaw/skills`
- Deployed skills: `${HOME}/.zeroclaw/workspace/skills`
- Config: `${HOME}/.zeroclaw/config.toml`
